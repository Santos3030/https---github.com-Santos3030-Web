Etiquetas de Documentos

Las etiquetas HTML son fragmentos de código que definen la estructura y el contenido de una página web, indicando al navegador cómo mostrar elementos como párrafos, encabezados, imágenes y enlaces. La mayoría vienen en pares de apertura (<tag>) y cierre (</tag>), aunque algunas son autocontenidas como <br>. Se utilizan para organizar el contenido y facilitar la interpretación del documento.



1.1 HTML

Las "tags" HTML, o "etiquetas" HTML, son códigos utilizados para "marcar" el texto de una página web, con el fin de dar instrucciones al navegador sobre cómo mostrarlo. Es decir, las etiquetas HTML son el lenguaje utilizado para estructurar y definir el contenido en un documento HTML. Estas etiquetas se encuentran en el HTML (o Lenguaje de Marcado de Hipertexto) de cada página.



1.2 head

Es un contenedor de metadatos sobre un documento HTML. Se coloca entre las < html>etiquetas `<head>` y ` <body></head>`. El contenido dentro de la <head>etiqueta `<meta>` no se muestra directamente en la página web, pero proporciona información al navegador y a los motores de búsqueda sobre el documento.



1.3 body

El elemento <body> de HTML representa el contenido de un documento HTML. Solo puede haber un elemento <body> en un documento.



Etiquetas para meta datos

son fragmentos de código <meta> que se colocan en la sección <head> del HTML de una página web para describir su contenido a navegadores y motores de búsqueda. Sirven para proporcionar información útil para el SEO, como la descripción, palabras clave y autor, así como para controlar el comportamiento de la página, la visualización en diferentes dispositivos y para solicitar la indexación o no de la misma.



2.1 meta

Se utiliza para proporcionar metadatos sobre un documento HTML. Los metadatos son datos sobre datos y, en el contexto de una página web, se refieren a información que describe el documento en sí, en lugar del contenido que se muestra en la página. Estos elementos se colocan dentro de la <head>sección `<head>` de un documento HTML y no son directamente visibles para los usuarios que visualizan la página.



2.2 style

Es el elemento encargado de indicar la información de estilo. Sus etiquetas son: <style> y </style> (ambas obligatorias). Está definido como: Elemento de cabecera. Crea una caja: No. Puede contener: Declaraciones de estilo. Debe ser contenido por: Elementos <head.



2.3 title

Se refiere al contenido dentro del <title > elemento `<h2>`, que se coloca dentro de la <head>sección `<head>` de un documento HTML. Este elemento define el título de la página web.



2.4 base

Especifica la dirección URL base que se utilizará para todas las direcciones URL relativas contenidas dentro de un documento. Sólo puede haber un elemento <base> en un documento.




2.5 link

Especifica la relación entre el documento actual y un recurso externo. Los usos posibles de este elemento incluyen la definición de un marco relacional para navegación. Este elemento es más frecuentemente usado para enlazar hojas de estilos.




Etiquetas de Agrupacion

Es lo que se llama un contenedor, es decir, una etiqueta que contiene un grupo de etiquetas relacionadas de alguna forma. Aunque no tiene que tener una clase de forma obligatoria, a las etiquetas <div> se les suele añadir una clase por varios motivos




3.1 P

Se usa para definir un párrafo de texto, organizando el contenido y añadiendo automáticamente un espacio antes y después de él. Se abre con <p> y se cierra con </p>, aunque la etiqueta de cierre es opcional en algunos casos. Es el elemento más apropiado para estructurar bloques de texto en una página web, mejorando la legibilidad y la accesibilidad.




3.2 hr

Se usa para crear una línea horizontal que separa temáticamente el contenido de una página web. Significa "horizontal rule" y es una etiqueta de autocierre que ayuda a organizar y estructurar el contenido, facilitando la lectura.




3.3 ul

Crea una lista desordenada, que se muestra típicamente con viñetas (como círculos o puntos) en lugar de números. Para cada elemento de la lista, se utiliza la etiqueta <li> (ítem de la lista) dentro de las etiquetas de apertura y cierre <ul> y </ul>.




3.4 div

Es un contenedor genérico que se utiliza para agrupar otros elementos HTML en una sección o bloque. Su principal propósito es estructurar el contenido de una página web para poder aplicarle estilos con CSS, aunque no tiene ningún significado semántico propio. Es una etiqueta de bloque que ocupa todo el ancho disponible y sirve para organizar secciones, crear diseños y manipular bloques de contenido.




3.5 figure

Se usa para agrupar contenido multimedia como imágenes, diagramas o fragmentos de código, que está relacionado con el flujo principal del documento, pero que puede ser movido sin afectar el significado principal del texto. Dentro de esta etiqueta, se utiliza la etiqueta <figcaption> para añadir una descripción o leyenda a ese contenido.




3.6 figcaption

Se usa para proporcionar una leyenda o pie de foto a un elemento <figure>. Su función es asociar texto descriptivo a contenido multimedia como imágenes, diagramas o videos, mejorando la accesibilidad y la estructura del contenido. Este texto se mostrará generalmente debajo del elemento visual dentro de la figura.




Etiquetas Textuales

Se utilizan para dar formato y estructura al texto, como <strong> para texto importante, <em> para texto enfatizado o cursiva, y <b> para texto en negrita sin importancia semántica. También existen etiquetas para saltos de línea (<br>), párrafos (<p>), y para crear líneas horizontales (<hr>), entre otras.




4.1 a

Se utiliza para crear hipervínculos, es decir, enlaces a otras páginas web, archivos, secciones dentro de la misma página, o direcciones de correo electrónico. Para crear un enlace, se utiliza la etiqueta junto con el atributo href, que especifica la dirección del destino, y el contenido del enlace, que es el texto o la imagen que el usuario verá y sobre el que hará clic.




4.2 span

Es un contenedor en línea genérico que agrupa elementos de texto u otros elementos en línea para aplicarles estilos (con CSS) o manipularlos con JavaScript. A diferencia de <div>, que es un elemento de bloque, <span> no tiene un efecto visual propio y solo se vuelve útil cuando se le aplica un estilo, permitiendo personalizar partes específicas de un párrafo o una oración sin afectar el flujo normal del texto.




4.3 sub

Se utiliza para representar texto en subíndice, haciendo que aparezca ligeramente por debajo de la línea de texto normal y con un tamaño de fuente más pequeño. Es ideal para fórmulas químicas, matemáticas o cualquier notación donde se necesite un texto en esta posición.




4.4 sup

Se usa para mostrar texto en superíndice, como exponentes en fórmulas matemáticas o notas al pie de página. El texto dentro de la etiqueta <sup> se muestra ligeramente más arriba y a menudo en un tamaño de fuente más pequeño que el texto normal. Por ejemplo, para escribir "\(10^{2}\)", se usaría <p>10<sup>2</sup></p>.




4.5 cite

Se usa para referenciar el título de una obra creativa (como un libro, película o canción) o el autor de un trabajo citado. Su uso es semántico, ayudando a los motores de búsqueda a entender el contenido, y el texto dentro de la etiqueta se muestra generalmente en cursiva por defecto en los navegadores.




Etiquetas Semanticas

Como <header>, <nav>, <main>, <article>, <section> y <footer>, definen el significado del contenido que engloban, en lugar de simplemente agruparlo como las etiquetas no semánticas como <div>. Son cruciales para la accesibilidad, la organización del código y el SEO porque ayudan a los navegadores, desarrolladores y motores de búsqueda a comprender la estructura y el propósito de cada parte de una página.




5.1 nav

Se utiliza para definir una sección de enlaces de navegación principales dentro de una página web, como el menú de inicio, de contacto o de servicios. Es una etiqueta semántica que ayuda a los navegadores, desarrolladores y lectores de pantalla a entender la estructura del sitio, mejorando la accesibilidad y la optimización para motores de búsqueda (SEO).




5.2 section

Se utiliza para agrupar contenido temático relacionado y crear secciones genéricas dentro de una página web. Es una etiqueta semántica que mejora la accesibilidad y el SEO al estructurar el documento de manera lógica, y se debe utilizar cuando existe un encabezado (como <h2>) para agrupar el contenido que le sigue.




5.3 article

Define un contenido independiente y autónomo, como una entrada de blog o un artículo de noticias, que podría ser distribuido o reutilizado por sí solo. Se utiliza para agrupar piezas de contenido que tienen sentido por sí mismas, como comentarios de un foro, o widgets interactivos.




5.4 header

Define el encabezado de una sección o documento, generalmente ubicado en la parte superior y que puede contener elementos como el logotipo, el título (usando <h1>) y el menú de navegación (<nav>). Esta etiqueta se utiliza para proporcionar información esencial sobre la página y va colocada dentro de la etiqueta <body>.




5.5 main

Se usa para definir el contenido principal de una página web y debe contener el contenido único y más relevante de la misma. Su propósito es ayudar a los navegadores, lectores de pantalla y motores de búsqueda a identificar el contenido principal, mejorando la accesibilidad y el SEO. Es importante que solo exista un elemento <main> por documento.




5.6 footer

Define el pie de página de un documento o una sección, conteniendo información complementaria como derechos de autor, enlaces a la política de privacidad, datos de contacto o enlaces a redes sociales. Su uso es semántico y proporciona estructura a la web, ayudando también a la accesibilidad y al SEO.
